# Input

The Input component is a wrapper around the native input element. It supports all the same attributes and events as the native input element.

## Usage

<Input />

```svelte
<script>
  import { Input } from "ziggyui-svelte";
</script>

<Input />
```

## Props

### `type`

The type of input. Defaults to `text`. Can be any of the following:

- `text`
- `email`
- `number`
- `password`
- `search`
- `tel`
- `url`

{" "}

<Input type="number" />

```svelte
<script>
  import { Input } from "ziggyui-svelte";
</script>

<Input type="number" />
```

The pattern to validate the input. Defaults to `""`.

### `hasError`

Whether the input has an error. Defaults to `false`. This will add a aria-invalid attribute to the input.

<Input type="number" hasError />

```svelte
<script>
  import { Input } from "ziggyui-svelte";
</script>

<Input type="number" hasError />
```

It is not required to add this prop directly if you are using a parent component to wrap the input. You can use [Svelte Context](https://svelte.dev/tutorial/context-api) to pass the error state to the input by setting `hasError`.

```svelte
<script lang="ts">
  import { setContext } from 'svelte';
  import { Input } from "ziggyui-svelte";

  setContext('hasError', true);
</script>

<div>
  <Input label="Checkbox" />
</div>
```

## Events
